PROJECT STRUCTURE EXPLANATION

This appears to be a structured Python data science / machine learning
project.

ROOT DIRECTORY FILES

LICENSE Defines how others are allowed to use your code (e.g., MIT,
Apache, GPL).

Makefile Contains command shortcuts for common tasks such as install,
test, lint, or training.

Prompt.txt Likely contains experiment prompts or configuration text for
models or LLMs.

pyproject.toml Modern Python project configuration file defining
metadata, dependencies, and build system.

README.md Project overview explaining what the project does and how to
use it.

------------------------------------------------------------------------

artifacts/ Stores outputs generated by experiments or pipelines.

features/ Saved feature matrices and engineered features.

figures/ Plots, charts, and visualizations.

models/ Trained model files (.pkl, .joblib, .pt, etc.).

runs/ Experiment outputs, logs, metrics, parameters.

------------------------------------------------------------------------

data/ Stores data across different processing stages.

external/ Third-party or downloaded datasets.

raw/ Original unmodified source data.

interim/ Cleaned but intermediate data.

processed/ Final datasets ready for modeling.

models/ Serialized preprocessing models such as scalers or encoders.

------------------------------------------------------------------------

mlruns/ MLflow tracking directory containing experiment metadata,
parameters, and metrics.

------------------------------------------------------------------------

notebooks/ Interactive experimentation area.

experiment1.ipynb Model experimentation notebook.

test.ipynb Testing or debugging notebook.

------------------------------------------------------------------------

RANDOM/ Scratch or experimental folder containing: - CSV datasets -
Scraping scripts - Portfolio analysis scripts

Likely contains temporary or exploratory work.

------------------------------------------------------------------------

src/ Main source code directory (best practice in Python packaging).

thesis/ Primary Python package containing core logic such as data
processing, modeling, CLI tools, and utilities.

thesis.egg-info/ Auto-generated package metadata.

------------------------------------------------------------------------

tests/ Unit and integration tests.

test_cli.py Tests command-line interface.

test_config.py Tests configuration loading.

test_math_utils.py Tests math helper functions.

test_smoke.py Basic functionality test.

pycache/ Auto-generated Python bytecode cache.

------------------------------------------------------------------------

Typical Workflow:

1.  Raw data -> data/raw
2.  Cleaning -> data/interim
3.  Feature engineering -> data/processed
4.  Training -> artifacts/models
5.  Evaluation plots -> artifacts/figures
6.  Tracking -> mlruns
7.  Core logic -> src/thesis
8.  Tests -> tests
9.  Exploration -> notebooks

Overall, this is a structured machine learning project designed for
reproducibility and experimentation, possibly for a thesis.
